## Architecture

This is a **Next.js 15 App Router** app for a logistics mission planner (drone + truck route optimization). The entire application lives at the `/map` route.

### State Management

All application state lives in `app/map/FlightPlannerContext.tsx`. This is a large React Context that holds:
- Mission nodes (orders, depot, stations, waypoints, hazards)
- Fleet configuration (drone/truck counts and specs)
- Computed routes returned from the backend
- UI state (active tabs, sidebar, selections)

Almost every component consumes this context ŌĆö understand it before modifying component behavior.

### Data Flow

1. User creates/edits `MissionSite` nodes in the UI
2. User triggers route generation ŌåÆ POST to `http://localhost:8000/api/routes/generate`
3. Backend returns `MissionConfig` with truck routes + drone sorties with timing
4. Map visualizes routes; Timeline/Gantt tabs generate displays from timing data

### Map Rendering

- **Leaflet** (not Mapbox) via `react-leaflet` ŌĆö all map components must be client-only
- Any component using Leaflet must be wrapped in `dynamic(..., { ssr: false })` or be a child of one
- `components/` holds reusable Leaflet primitives: `ArrowheadPolyline`, `ArrowheadArcPolyline`, `NumberedMarker`, `TextMarker`
- `MissionSiteMarker.tsx` renders different marker types based on `MissionSite.type`

### Visualization Tabs

- `gantt/` ŌĆö Gantt chart for driver/drone schedules, data generated by `useGanttData.ts`
- `timeline/` ŌĆö Mission event timeline, data generated by `useTimelineGenerator.ts`
- `routes/` ŌĆö Route details; `useRouteDetails.ts`
- `tables/` ŌĆö Editable data tables for mission sites, orders, routes, vehicles

### Styling

- **Tailwind CSS 4** (using `@tailwindcss/postcss`)
- **Radix UI Themes** wraps the root layout ŌĆö use Radix components for UI primitives
- **Lucide React** for icons
- Prettier config: 125 char line width, single quotes, trailing commas
